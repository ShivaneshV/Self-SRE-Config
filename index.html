1. What is Self.SRE? (Project Brief)

When a production server crashes, SRE systems instantly reroute traffic and page developers. But when a human engineer crashes (due to migraine, burnout, or emergencies), our schedules fall into chaos.

Self.SRE treats the human developer as a production node, and their daily calendar configuration (schedule.json) as version-controlled code in a GitLab repository. Using Google Cloud Agent Builder and the GitLab MCP, a Gemini AI Daemon continuously monitors biometric feeds and acts as an autonomous site reliability manager.

When an incident spike occurs, the daemon initiates a GitLab hotfix branch, commits schedule revisions, and opens a Merge Request. Once approved, local webhook handlers execute real-world routing (Slack OOO indicators, calendar blocks, and client email cancellations) to clear the node for recovery.

3. GitLab Integration & Flow

  • Telemetry Monitor: The daemon polls biometrics (Heart Rate, Stress index).
  • Linter Audits: The GitLab CI/CD runner parses new commits to catch cognitive overload storms.
  • MR Failover Gate: If telemetry spikes, Gemini opens an MR, waiting for human approval.
  • API Failover Execution: Merging triggers the webhook, wiping calendar nodes and spinning up the digital twin proxy.
