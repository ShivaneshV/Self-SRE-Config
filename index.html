Operations Manual: Self.SRE

1. Core Architecture

Self.SRE applies Enterprise DevOps to human biology. Your schedule is stored as schedule.json in a GitLab repository. A Gemini Daemon continuously monitors biometric telemetry. If a node failure occurs (burnout, panic attack), it autonomously creates a GitLab Branch, drafts a Merge Request, and triggers physical failovers (canceling meetings, deploying AI Proxies).

2. Chaos Monkey Testing

Use the Chaos Monkey Incident dropdown in the top right to simulate disaster scenarios:

  • Sev-1 Burnout Spike: Triggers the Dead Man's Switch, spikes the HTML5 Canvas ECG, and executes an automated failover. Displays the Blameless Postmortem.
  • Context-Switch Storm: Switches to the Bio-Linter tab and fails the CI/CD pipeline due to cognitive overload.
  • SLA Sleep Breach: Drops heart rate to recovery levels and issues an SLA warning for sleep deficit.

3. Real-Time API Sync

This dashboard features asynchronous polling. When run locally alongside sre_webhook_receiver.py, it syncs terminal logs into the UI in real-time. You can trigger incidents using the simulate_incident.py CLI tool, and the web UI will react automatically without manual clicks.
